Appendix C: Correction Tables for Angle Correction in Case of Inclinometer Measurements

Hint

Creating correction tables for angle correction is the sole responsibility of the user.

A correction table for angle correction can be created by means of a spreadsheet program (e.g., Microsoft Excel) or an editor.

Use tabs as field separators and save a correction table as a tab separated text file with the extension *.cot.
 

A correction table must have the following structure (for this, refer also to Fig. Example of a correction table for angle correction at an OCTAVIUS Detector 729):

  • -Row 1 must always indicate the title "PTW Angle Correction Table".
  • -Row 2 must always indicate a ptwDetectorKey. The following ptwDetectorKeys are allowed:
    • -Octavius_729
    • -Octavius_729XDR
    • -Octavius_1000
    • -Octavius_1500
    • -Octavius_1500XDR
    • -Octavius_1500MR
    • -2DArray_729
    • -2DArray_729XDR
Hint

The correction tables from VeriSoft < 6.0 that have been created by means of PTWDeviceKeys can still be used.

  • -Rows 3 and 4 can contain user-defined entries or they can be empty.
  • -Row 5 up to row x indicate the correction values.
    • -For each column of the detector array, a column must be created in the correction table. The column name shows the distance between the measuring chamber and the center chamber in negative or positive direction. You will find the column names for the respective detector arrays in the following table.
    • -Row 6 up to row x indicate the correction values for the desired angle ranges. A correction value is always valid for one column of the detector array. The angle ranges can be defined by the user. You can set at most 360 angle ranges.
      Example:

      The same correction value, entered for 30°, will be assigned to all angles from 15° < x ≤ 30°. An interpolation will not be carried out.
