Supported Data Formats

VeriSoft is able to read matrices with the following formats:

PTW Formats:

  • -detector array data (*.mcc)
    This includes also:
    • -merged detector array data
    • -composed detector array data
    • -sections of detector array data (ROI)
  • -OCTAVIUS 4D data (*.xcc)
  • -detector array data with gantry angles (*.xcc)
  • -PTW mcc format (*.mcc)
  • -MEPHYSTO 7 export format (*.exp)
  • -FIPS Plus images (*.img)
  • -MEPHYSTO ASCII-matrices (*.dat)
  • -PTW image format (*.dat)

Graphic Formats:

  • -TIFF (*.tif)

Dose Matrices of Different Planning Systems:

Only the planning systems requiring special procedures are listed below.

If your planning system is able to export DICOM RTDose objects, you should always select this function for exporting.
Furthermore, you should keep in mind that you have to export the slice of dose volume that is lying in the effective measuring point of the detector array.
You can also export a complete dose volume. In this case, you can select the desired slice by means of the slider in the pop-up window Slice.

  • -DICOM RTDose (*.dcm)
  • -BrainLab BrainScan 5.x (*.axl, *.cor, *.sag, *.dat, *.flu)
  • -Nucletron PLATO Dose Grid File RTS Version 2.1 + 2.5 (*.d*)
    For this file, a simulation file (*.s*) must exist in the same directory.
  • -CMS Focus RTP QA Dose Output File (*.txt)
  • -Varian Cadplan > 6.2 (*.*ef*)
    For these files, CART files must exist in the same directory (identical file names whose name extension is an index).
  • -Philips Pinnacle3 Version 6.x (*.header) or
    (* / Ascii file without file name extension)
    For each *.header file, an image file (*.img) must exist in the same directory.
  • -Leibinger Virtuoso (*.hed)
    For this file, a dose file (*.sdc or *.dos) must exist in the same directory.
  • -Radionics 2D Table (*.smo)
  • -3D Line ERGO (*.exp)
  • -DICOM RTImage (*.dcm)
  • -TomoTherapy Hi•Art Export (*.header)
    For each *.header file, an image file (*.img) must exist in the same directory.
  • -K&S Diamond (*.txt) Version 5.0 or higher

The version numbers stated refer to the date the user manual was created. If in doubt, please contact PTW-Freiburg for information about the compatibility with newer versions.
