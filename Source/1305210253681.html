Settings for Importing DICOM Data

DICOM tab of the Options dialog

On the DICOM tab, you will find the following options:

Multi Frame Images

In this pane, you determine the slice direction of the loaded dose volume to be calculated. To do so, you select the desired direction in the list box. If the selection and the saved slice direction are identical, the data will be loaded unchanged.
Default: Transversal

Data Import

If you want to import the DICOM data directly from the DICOM network server, you have to adjust the following parameters:

  • In the Remote AESettings pane (AE = Application Entity), you have to adjust the network parameters of the DICOM server. Your network administrator will provide these network parameters.
  • -SCP-Server name:
    computer name of the server
  • -Unique AE title:
    unique AE title of the server
  • -TCP/IP port:
    port of the server for accessing the DICOM SCP (SCP = Service Class Provider)
  • In the Local AESettings pane (AE = Application Entity), you have to adjust the network parameters of your PC, i.e., of the DICOM client. Normally your network administrator has to adjust the network parameters of the client also at the DICOM server, i.e., the client has to be made known to the server.
  • -Unique AE title:
    unique AE title of the client
  • -TCP/IP port:
    port of the client for accessing the DICOM SCP (SCP = Service Class Provider)

Windowing

Select the Apply VOI/LUT Transformation check box to instruct VeriSoft to include the VOI/LUT transformation saved in the DICOM file when loading a matrix.
